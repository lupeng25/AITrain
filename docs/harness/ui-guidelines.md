# UI Guidelines

## 设计方向

AITrain Studio 的 UI 定位是本地视觉训练工作台，不是营销页。

参考方向：

- Linear：清晰的信息架构。
- NVIDIA：GPU / 训练平台的技术状态表达。
- Airtable：数据集、插件、任务表格。
- Cursor：日志和开发者工具式反馈。

## 布局

当前采用：

```text
左侧导航 + 顶部状态栏 + 中央工作区 + 底部状态栏
```

不要回退到 `QTabWidget` 原型结构。

## 组件约束

- 页面级容器用 `InfoPanel`。
- 状态标签用 `StatusPill`。
- 导航用 `Sidebar`。
- 全局样式放在 `AppStyle`。
- 图表继续通过 `MetricsWidget` 或后续专用图表组件实现。

## 文本

- UI 文本使用中文。
- 使用 `QStringLiteral`。
- 不使用乱码或拼音占位。
- scaffold 能力必须标注为“占位”或“后续接入”。

## 训练页

训练页必须保留：

- 配置区。
- 进度条。
- 指标曲线。
- 日志。
- 任务/产物说明。
- 启动和取消入口。

## 表格

表格必须：

- 有明确表头。
- 禁止编辑，除非是表单表格。
- 选中整行。
- 隐藏垂直表头。
- 支持空状态说明。
